# API Automation Testing Framework for QA Automation Sandbox

Данный репозиторий содержит фреймворк автоматизации тестирования API для обучающей платформы **QA Automation Sandbox** (оригинальный репозиторий приложения: [qa-automation-sandbox](https://github.com/manikosto/qa-automation-sandbox)).

Фреймворк реализует модульное, интеграционное и e2e-тестирование бэкенда, эмулируя поведение различных типов пользователей (администраторы, модераторы, приватные аккаунты, забаненные и незарегистрированные пользователи).

---

## Технологический стек

* **Язык программирования:** Python 3.10+
* **Тестовый фреймворк:** `pytest`
* **HTTP-клиент:** `requests`
* **Property-Based Тестирование:** `hypothesis`
* **Генерация тестовых данных:** `faker`

---

## Структура проекта

```text
SANDBOX/
├── api_frame/              # Основной пакет взаимодействия с API (API Objects)
│   ├── .env                # Переменные окружения (конфигурация)
│   ├── auth.py             # Эндпоинты авторизации, регистрации и работы с токенами
│   ├── base_api.py         # Базовый класс для конфигурации HTTP-запросов и сессий
│   ├── follows.py          # Эндпоинты подписок, отписок и обработки запросов
│   ├── posts.py            # Эндпоинты для работы с публикациями
│   └── users.py            # Эндпоинты профилей пользователей и списков
├── helpers/                # Вспомогательные утилиты (утилитарный слой)
│   ├── assertions.py       # Кастомные хелперы для гибких проверок ответов сервера
│   └── data_generator.py   # Генератор случайных валидных профилей на базе Faker
├── tests/                  # Набор автоматизированных тест-кейсов
│   ├── test_auth/          # Тестирование аутентификации, сессий и регистрации
│   │   ├── test_get_me.py  # Проверка получения информации о текущем пользователе
│   │   ├── test_login.py   # Сценарии логина (параметризованные и Property-Based тесты)
│   │   └── test_reg.py     # Тесты создания учетных записей (регистрация)
│   ├── test_follows/       # Тестирование социальной логики
│   │   └── test_user_follows.py # Интеграционные multi-session тесты подписок/приватности
│   └── test_user/          # Тестирование работы с пользователями
│       └── test_user.py    # Проверки обновления профилей и получения списков
├── conftest.py             # Pytest-фикстуры (фабрика токенов, создание сессий)
├── pytest.ini              # Конфигурационный файл pytest (настройка pythonpath)
├── requirements.txt        # Список зависимостей проекта
└── .gitignore              # Исключение кэша, виртуального окружения и IDE-файлов
```

## 1. Подготовка окружения
Склонируйте репозиторий, настройте виртуальное окружение и установите необходимые зависимости: 
### Клонирование репозитория
```text
git clone [https://github.com/manikosto/qa-automation-sandbox.git](https://github.com/manikosto/qa-automation-sandbox.git)
cd qa-automation-sandbox
``` 

### Создание виртуального окружения
```text
python3 -m venv venv
```

### Активация виртуального окружения (для Linux / macOS)
```text
source venv/bin/activate
```

### Активация виртуального окружения (для Windows)
```text
venv\Scripts\activate
```

### Установка всех необходимых зависимостей
```text
pip install -r requirements.txt
```

## 2. Запуск автотестов
Перед стартом убедитесь, что локальный сервер приложения QA Automation Sandbox запущен и доступен по адресу http://localhost:8000 (настройка по умолчанию зашита в base_api.py). 

### Запуск абсолютно всех тестов в проекте
```text
pytest
```

### Запуск тестов с подробным выводом логов и принтов в консоль (включая шаги Hypothesis)
```text
pytest -v -s
```

### Запуск конкретной папки с тестами (например, тесты социальной логики подписок)
```text
pytest tests/test_follows/
```

### Запуск одного конкретного тест-файла
```text
pytest tests/test_auth/test_login.py
```
